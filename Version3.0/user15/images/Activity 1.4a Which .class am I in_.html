    Activity 1.4a Which .class am I in?

    • Due No Due Date
    • Points 19







     

    What you will accomplish

    • Use 1 CSS file to control how many different HTML files are styled

    Introduction

    Many things in our world are classified, as people we are classified in the kingdom - Animalia, Phylum - Chorodata,  Class - Mammalia , Order - Primates, Suborder - Haplorhini, Infraorder - Similformes, Family - Hominidae, Subfamily - Homininae, Triber - Hominini, Genus - Homo, Type Species - Homo Sapiens.   Good thing it's a little bit easier than that when designing a web page.  We will look at how to create and assign CSS properties and how to organize those properties so we can use them in many areas of our webpage.  

    Materials

    • GitHub Desktop software
    • GitHub Account
    • Visual Studio Code
    • Internet Browser (Google Chrome)

    Procedure

    Part I: Using a class definition

    1. Open your Version 4.0 folder and select index.html file

    2. Copy and paste the code from this repository (Links to an external site.) into your index.html file. After you paste the code Fix the formatting by pressing (Shift+Option+F)

    3. Open the style.css file from your CSS folder in Version 4.0 

    4. Copy and paste the code from this repository (Links to an external site.) into your style.css file. After you paste the code Fix the formatting by pressing (Shift+Option+F)

    5. Verify that you now have your .CSS and .html files in your Version 4.0 folder. by opening up your index.html in a chrome browser, it should look like this. 

    6. We will look at buttons in this lesson and see how we style the same elements differently using the same external .CSS file. 

      • The lines of CSS code are here to help us style a button.  BUT that's not all, we can also use the class definitions to style anything else! more on that later.
      • If we want to have more than 1 button on our web page AND we want those buttons to look different we can define different classes. In this case we define a basic button giving it a background color, font size and other properties.  Then we can make more classes that will help us further define what that button needs to look like.  

    7. A class selector in CSS starts with a dot (.), like this: 

    1
2
3
    		 
    .class{
      property: value;
    }

    8. A "CSS class", is styling information that is meant to be used on multiple elements.  You can name them whatever you like, however when we name classes we have the opportunity to help us organize and communicate our code better by giving them names that mean something.  Calling a class ".red" makes sense if the styling information includes changing a color to red.  However ".button2" is not a good name because users will always have to check to see which styling properties this class carries.  

    Classes can be used on many different elements.  the ".red" class can be called on a <h1>, <p>, <div>, <table> and the list goes on.  That is what makes them powerful - in their versatility and customizability. In the next section we will learn about a language that is based on CSS selectors alone and most of the language focuses on classes and sub-classes.  

    9. In our index.html file you will see 5 of these buttons.  These buttons are very similar in how they are coded in our HTML file, the difference is which class is being applied.  Notice that button 3, 4, and 5 all use multiple classes. <button class="button button2 red">

    10. In your index.html file, add 3 more buttons. (Make sure to delete the green comment)

    1
2
3
    		 
    <td>
      <button>Text you want on your button</button>
    </td>

     

    11. In your style.css create 3 new classes to be used with your 3 new buttons.  (Class names cannot begin with numbers.)

    • Challenge yourself here.  Try and find more than just changing the color or text size, there are many different properties to choose from that will transform your buttons. Use these resources to help you find some ways to change your buttons. 
    • https://www.bestcssbuttongenerator.com/ (Links to an external site.)
    • https://tympanus.net/Development/CreativeButtons/  (Links to an external site.)Link to CSS (Links to an external site.)
    • https://www.bitdegree.org/learn/css-button (Links to an external site.)

    Part II: Organizing Behaviors With Sub-Classes

    12. When we are writing code for our website it is in our best interest to make code that is reusable.  This will allow for us to easily make websites down the line. AND we can easily make changes simply by creating a sub-class.

    13. Sub-class is just a novel way to say, “use another class name” when you recognize or plan reusable design patterns that relate to a parent class. If we use sub-classes we can move and reuse stuff in different places without breaking styles or rewriting the selectors.

    The key is to recognize reusable design patterns in our CSS, then write a subclass for it.  In this case we want all of our titles of things on our website to be the same color and font, BUT we want the size to be different.  

    (NOTE: Feel free to change the font, here is a link to a good resource on choosing fonts (Links to an external site.))

    ADD this code to your style.css file

    1
2

    .title{
      color: red;
      font-family: 'Trebuchet MS', sans-serif; 
    }

    .title-large {
      font-size: 2rem;
    }

    .title-small {
      font-size: .5rem;
    }

    14. When you refresh your browser you will notice that each item with the class "title" has turned red.  Let's modify our html to also include the sub class .title-large and .title-small.

    15. In your index.html file ADD the subclass title large and title small to their respective lines of code.  Below is what they should look like when you are complete. 

    1
    2
    		 
    <span class="title title-large">Example of a Heading Title</span>
    <span class="title title-small">Example of a Footer Title</span>

    16. Since "title" uses CSS subclasses, we can move it and reuse it anywhere on the page without broken styles.

    17. Update 2 of your buttons to use different sub-classes.  (A good example would be to eliminate button1 and button2 and instead call them something like - button-largeRadius or button-smallRadius.) Here is a resource (Links to an external site.)

    Part III: ID# selectors

    18. We will now look at another way to target an element with styling using the #id selector.  The #id selector is in many ways like a class. Meaning - we can call it up like this in our external CSS file or we can call it internally in the HTML.

    1
2
3
    		 
    #morecontent {
      background-color: yellow;
    }

    19. The #id selector is meant to be only used once, even though it can be used many times- it is best practice to make a class if you want to use it more than once.  Go ahead and place this code into your CSS file and note the changes on your web page. 

    20. This example above is not a great way to use an ID because "more content" can happen all over the place and many times on your webpage.  A better use of ID's are found on the lines below. 

    1
2
3
    		 
    <table id="hireButtons">

    <table id="footerTable">

    21. There should only be one part of your webpage that has hire buttons and since there is only 1 "footer", you probably won't have any more than 1 table that exists inside the footer. 

    22.  Brainstorm some styling you can call on your button table area of your webpage and some styling you can call on the table in your footer.  Then in your style.css file write a styling definition for both the id hireButtons, and the id footerTable.  

    Part IV: Universal selector

    23. Sometimes we want FULL CONTROL!!!  We can change EVERY ELEMENT on our webpage by using the universal selector.  While typically it is not always in our best interest to apply styling to all elements this is how you would do it. Add these lines to your sampleCSS.css file

    1
2
3
    4
    		 
    *{
        text-align:right;
        border: 2px solid black;
    }

    25.  Your webpage turns pretty ugly, but you now know a way to change every element on your webpage.  Maybe it's St. Patrick's day and you want to change all your font to green color #00ff00, using the universal selector, *, would be an easy way to have an effect on all your elements. 

    26. Now it's your turn to think of something you would want to add to your webpage through a universal selector. In your style.css file remove the text align and border and add your own styling. 

    Part V: Saving Changes

    For detailed instructions refer to Assignment 1.1a Tools for Collaboration

    27. Although your changes are saved on your local computer, they are not stored permanently by git. To create a permanent checkpoint of this change, you need to make a commit. Enter a summary and description then select Commit to master   

    28. You need to sync your local changes to the remote repository on GitHub. Since your changes are only on the branch, it will ask to push to origin.  Click on the "Push to Origin" button in the upper right hand corner. This will send your commit up to the remote repository.  If it does not say push to origin then there is nothing new push.
